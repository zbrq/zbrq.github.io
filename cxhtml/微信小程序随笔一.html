帮助注册微信小程序，配置微信小程序全局设置，设置微信小程序全局样式的4个文件
        app.js   帮助注册微信小程序应用   
        app.json  帮助进行微信小程序的全局配置(小程序全局配置文件)，包括网络请求超时时间，窗口的表现，以及各个页面的注册路径
        app.wxss    帮助设置微信小程序的全局样式
        project.config.json     保存微信开发者工具的配置信息，针对微信开发者使用微信开发工具的设置爱好，记录开发者对开发工具的设置《项目的个性化设置，根据开发者自己的配置，改变界面颜色，调试工具的位置》

-----------------------------------
pages文件夹：  帮助存放自己微信小程序的所有的小程序页面
    每个小程序页面最多是由4个文件组成《js文件，json文件，wxml文件，wxss文件》
--------------------------------------
4个必不可少的基础文件

    js文件 ： 帮助处理微信小程序页面的逻辑和数据交互
    json文件： 帮助处理微信小程序页面的配置信息
    wxml文件: 帮助展示小程序页面的一些元素和内容
    wxss文件： 帮助设置小程序页面的样式

-------------------------------------------------------
utils.js文件 ：  存放一些工具的函数，以达到代码复用的目的

全局配置有五项：

Pages       注册微信小程序的所有页面
tabBar      如果微信小程序是一个多table按钮的程序就可以通过这个tabBar的配置来指定table栏的表现
networkTimeout      设置各个网络请求他的一个超时时间
debug           开启debug，来在控制台打印出我们需要的调试信息
navigationStyle

----------------------------------------------
通过enablePullDownRefresh 设置
导航栏的设置，窗体颜色背景等样式设置，窗体下拉时的字体样式，单个页面下拉或全局页面下拉刷新的表现


disableScroll 设置页面是否开启滚动


如果页面的配置和全局配置重复了，页面配置会覆盖掉全局配置

创建第一个微信小程序

    首先存在一个project.config.json文件
基础文件
1.创建app.js文件
    调用APP函数<可以传入一些生命周期钩子和一些事件处理函数还有全局的数据>
        App({})

2.创建app.json文件
    用于注册我们的微信小程序所有页面的路径(通过pages属性注册，pages的值是list类型的值，list里面包含的是所有页面的一个页面路径)
    {
        "pages": []
    }

创建自己的目录文件 《注意创建的这4个文件命名需要保持一致》
3.创建XXXX.wxml 文件
                ---》相当于html文件,页面展现文件
    <view>hellow world</view>

4.创建XXXX.js  文件
                ---》帮助注册微信小程序页面，作用类似app.js
                ---》通过配置函数帮助注册，配置函数需要传入一个参数,参数类型是object类型
                ---》参数可以传入页面的生命周期钩子和事件处理函数，以及页面的一些默认数据
    Page({})


5.创建XXXX.json  文件
                --》是wxml页面的一些配置，至少要传入一个空的配置对象


    {}

6.创建XXXX.wxss  文件
                ---》用于描述xxxx.wxml文件的样式，相当于css文件的作用

    view{
        width:100%;
        height:40rpx;   
        text-align:center;
    }

7.在app.json 文件中注册xxxx文件的路径

    {
        "pages": [
            "xxxx"   //在根目录下直接填文件名，若有文件夹则要填写正确路径
        ]
    }
8.找到文件目录，进去新建first目录，把xxxx4个文件放进first目录，到app.json去更改文件注册路径

    {
        "pages": [
            "pages/first/xxxx"   //在根目录下直接填文件名，若有文件夹则要填写正确路径
        ]
    }

小程序的开发框架

小程序由4个部分组成：
    wxml    页面展现
    wxss    页面样式
    wxs     对wxml能力增强的脚本语言，可以把请求得来的数据进行计算处理，帮助wxml快速构建出页面的内容结构
    javascript  处理页面交互逻辑和数据通讯

WXML（Weixin Markup Language）

wxml是框架设计的一套标签语言,结合组件、wxs和事件系统，可以构建出页面结构。
语法：大小写敏感

<标签名 属性名="属性值1" 属性名="属性值2"...> </标签名>

wxml的4个特性: 数据绑定，列表渲染，条件渲染，模板引用

-----数据绑定

===》数据绑定:
1.  文本内容绑定
    <!--index.wxml-->   
    <view>
        <text>{{message}}</text> //变量用{{}}括起
    </view>

    //index.js
    Page({
        data: {
            message:"Hellow,world"
        }
    })
------------------------------------
2.  绑定属性！
    <!--index.wxml-->   
    <view>
        <text data-name="{{abc}}"> </text> //变量用{{}}括起
    </view>

    //index.js
    Page({
        data: {
            abc:"sssssss"
        }
    })

    结果显示为：
    <view>
        <text data-name="sssssss"> </text> 
    </view>

----------------------------
3.  运算符绑定
    <view hidden="{{flag ? true : false}}">
        Hidden 
    </view>

    //index.js
    Page({
        data:{
            flag:false          
        }
    })

其结果：false  显示Hidden这个单词 ，如果为true则必定隐藏掉Hidden

属性名 类型 描述 注释

----列表渲染

<!--index.wxml--> 列表例子
<view>
    <block wx:for="{{items}}" wx:for-item="item" wx:key="index">
        <view>{{index}}:{{item.name}}</view>
    </block>
</view>
//index.js
Page({

    data:{
        item:[
            {name:"111111"},
            {name:"222222"},
            {name:"333333"},
            {name:"555555"},
            {name:"666666"}

        ]

    }
})

结果：  1111111
       2222222
       3333333
       5555555
       6666666

----条件渲染

<!--index.wxml--> 条件渲染例子
<view>今天吃什么?</view>

<view wx:if="{{condition === 1}}">饺子</view>
<view wx:elif="{{condition === 2}}">米饭</view>
<view wx:else>面食</view>

//index.js
Page({

    data:{
            data:{
                condition:Math.floor(Math.random()*3+1)
            }
        ]

    }
})

结果：  今天吃什么?
        面食

----模板引用

文件引用： import 和 include

import引用例子：
<!--index.wxml-->
    <import src = "a.wxml"></import>//import引用
    <template is="a"></template>//引用的模板名

<!--a.wxml-->
    <view>Hello,world</view>   //没生效
    <template name ="a"> Hellow,World!</template> //被引用了


结果：Hello,World!

include引用例子：
<!--index.wxml-->
    <include src = "a.wxml">//include引用
    <template is="a"></template>//引用的模板名

<!--a.wxml-->

    <template name ="a"> 
        <view>Hello,world</view>   //没生效!
    </template>

    <view>SSSSSSSSSSS</view>  //被引用了


结果：SSSSSSSSSSS

wxss 选择器

选择器 样例 样例描述
.class .intro 选择所有拥有class="intro"的组件
#id #firstname 选择拥有id="firstname"的组件
element view 选择所有的view组件
element,element view,checkbox 选择所欲文档的view组件和是所有的checkbox组件
::after view::after 在view组件后边插入内容
::before view::before 在view组件前边插入内容

优先级：

!important 无穷大最高

!important > style > #element >.element > element


例子: 如何使用important
/**index.wxss**/
    .title{
        color:red !important;
    }

微信小程序JavaScript与普通JavaScript的不同之处

浏览器中的JavaScript
    ECMAScript      DOM     BOM

node.js中的JavaScript
    ECMAScript      Native      NPM

微信小程序中的JavaScript
    ECMAScript      小程序框架   小程序API


ECMAScript包含
    语法  类型  语句  关键字 操作符 对象

android====》X5内核
